# cgkit 粗粒化工作流介绍说明

## 1. 工具定位与核心目标

**cgkit**（Coarse-Graining Kit）是一套面向聚乙烯（PE）分子动力学模拟的端到端数据处理工具，用于将原子级（All-Atom, AA）的 LAMMPS 模拟轨迹转化为**粗粒化（Coarse-Grained, CG）**训练数据，进而服务于 DeepMD-kit 深度势能模型的训练。其核心目标是：

> **在全原子精度与粗粒化效率之间建立可逆、可控的数据桥梁**，从海量 MD 轨迹中筛选出最具代表性的构型子集，构建高质量、低冗余的训练集。

---

## 2. 整体工作流概览

整个工作流呈**线性递进**结构，由 7 个核心步骤组成，前 3 步为数据生产，后 4 步为质量评估与结构筛选：

```
原子级 LAMMPS 轨迹 → 粗粒化数据 → DeepMD 训练格式 → 温度参数
       ↓                     ↓              ↓              ↓
   结构分析（SOAP/PCA/聚类）→ P-T 覆盖评估 → 代表性构型筛选 → 子集输出
```

| 步骤 | 子命令 | 核心作用 |
|:---|:---|:---|
| **数据生成** | `cg-gen` | 将原子轨迹按化学规则映射为 CG 粒子（bead） |
| **格式转换** | `to-deepmd` | 将 CG 数据转为 DeepMD-kit 的 `.raw` / `.npy` 格式 |
| **温度标注** | `fparam extract/const` | 为每帧附加温度参数，支持变温/恒温模拟 |
| **统计概览** | `analyze-cg` | 对 CG 数据进行能量、粒子数、时间序列的统计分析 |
| **结构表征** | `analyze-atomic` | 通过 SOAP 描述符 → PCA → t-SNE → 聚类，量化构型多样性 |
| **覆盖评估** | `plot-pt` | 绘制 P-T 散点图，检查训练集在压力-温度空间的覆盖度 |
| **结构筛选** | `select-structures` | 在降维投影空间上聚类，用 maximin 采样挑出最具代表性的构型 |

## 3. 核心原理详解

### 3.1 粗粒化映射（Coarse-Graining）

这是整个工作流的**基石**。PE 分子由碳链（C）和氢（H）组成，全原子模型计算量巨大。粗粒化的基本思想是**将化学上紧邻的基团折叠为一个超原子（bead）**。

- **CH₂ 基团**：1 个 C + 2 个 H → 合并为 1 个 type-1 bead
- **CH₃ 端基**：1 个 C + 3 个 H → 合并为 1 个 type-2 bead

**算法采用两阶段策略**：
1. **手动指派（优先）**：对链端 CH₃ 等特殊基团，通过 `cg_assignments` 显式锁定原子 ID，避免被错误匹配。
2. **模式匹配（自动）**：对剩余碳原子，按 `[1,2,2]` 模式（1C+2H）在周期边界内寻找最近邻氢原子，动态归并为 bead。

每个 bead 的属性（位置、力、势能）默认取组内原子的**算术平均**，从而保留统计力学上的等效性。

### 3.2 从结构到向量：SOAP 描述符与高维降维

为了评估海量构型的"多样性"，需要一种与坐标系无关的**结构指纹**。

- **SOAP（Smooth Overlap of Atomic Positions）**：将局部原子环境编码为固定长度的向量，具有旋转、平移、置换不变性。
- **PCA**：将高维 SOAP 向量投影到正交主成分空间，压缩冗余信息。
- **t-SNE**：在低维空间（通常 2D）非线性地保持局部相似性，便于可视化聚类。
- **聚类（DBSCAN / KMeans）**：在投影空间中将相似构型归为一类，暴露结构族群的分布。

本质上，这一步是将"构型相似性"问题转化为**向量空间中的距离问题**。

### 3.3 代表性筛选：Maximin 最远点采样

训练集并非越大越好——相似的构型对模型提升的边际贡献极低。因此需要在每个聚类簇中挑选**彼此最分散的 N 个构型**。

**Maximin 策略**：
1. 第一个点选离簇质心最近的点（确定性锚点）。
2. 后续每一步，选择"到已选点集最小距离"最大的那个点。
3. 重复直至选满 N 个。

这保证了选出的构型在结构空间中**覆盖度最大化**，避免冗余。

### 3.4 P-T 空间覆盖评估

深度势能模型的泛化能力直接受训练集的相空间覆盖度影响。`plot-pt` 将每个 dump 帧的 timestep 关联到 LAMMPS 日志中的热力学输出，得到实测温度与压力，绘制 P-T 散点图。通过目视检查可快速发现：
- 温度/压力采样盲区
- 目标工况是否被充分覆盖
- 变温模拟中的升温/降温路径是否完整

## 4. 设计思路与架构特点

### 4.1 统一入口 + 配置驱动

- 以 `cgkit.py` 为单一 CLI 入口，替代原先 5 个独立脚本（约 5200 行）。
- 通过 `config.json` 集中管理所有路径与参数，命令行参数可即时覆盖对应配置键。

### 4.2 延迟加载（Lazy-Load）

- 核心流程（`cg-gen`, `to-deepmd`）仅依赖 `numpy`、`pandas`、`tqdm`。
- 重型依赖（`matplotlib`、`scikit-learn`、`torch`、`dscribe`）仅在调用分析子命令时按需导入，避免无意义的加载开销。

### 4.3 分析-筛选解耦

- `analyze-atomic` 一次性产出 `pca_results.csv` / `tsne_results.csv`。
- `select-structures` 可反复消费这些 CSV，尝试不同聚类数、采样数，无需重跑昂贵的 SOAP/PCA 流程。

## 5. 典型使用场景

| 场景 | 执行流程 |
|:---|:---|
| **标准 PE 粗粒化训练集构建** | `cg-gen` → `to-deepmd` → `fparam extract` → `analyze-cg` |
| **训练集结构多样性评估** | `analyze-atomic --mode cg` → `plot-pt` |
| **从大数据中筛选代表性子集** | `analyze-atomic` → `select-structures --n 3` |
| **变温/恒温模拟温度标注** | `fparam extract`（变温）或 `fparam const`（恒温） |

---

## 6. 总结

cgkit 的工作流本质上是一个**"降维-聚类-采样"**的闭环：

1. **降维**：从原子坐标到 CG bead，再到 SOAP 描述符，再到 PCA/t-SNE 低维投影，层层抽象。
2. **聚类**：在低维空间发现结构族群的内在分布。
3. **采样**：在每个族群中挑选最远点，确保覆盖度与多样性。

这套流程不仅适用于 PE 体系，其粗粒化映射规则、描述符计算、降维聚类与 maximin 筛选框架可推广至其他聚合物或软物质体系，只需调整 `config.json` 中的化学映射模式与模拟参数即可。
